# FOTX - Padrão de Commits Laravel/PHP (Português-BR)

Configuração específica para sugestões de commits do projeto FOTX, seguindo o padrão Conventional Commits em português-brasileiro.

## Formato Padrão

```
<tipo>(<escopo>): <assunto>

<corpo>

<rodapé>
```

## Tipos de Commit (Convencionais)

- **feat**: Nova funcionalidade ou melhoria
- **fix**: Correção de bug
- **refactor**: Refatoração de código sem mudança de funcionalidade
- **perf**: Otimização de performance
- **test**: Testes novos ou modificados
- **docs**: Alterações em documentação
- **style**: Formatação, espaçamento, sem lógica
- **chore**: Atualização de dependências, build, scripts
- **ci**: Alterações em pipeline/CI-CD

## Escopos Específicos (FOTX)

### Modelos & Banco de Dados
- `modelo-usuario` — Model `User`
- `modelo-evento` — Model `Event`
- `modelo-foto-evento` — Model `EventPhoto`
- `modelo-rosto` — Model `PhotoFace`
- `modelo-pedido` — Model `Order`, `OrderItem`
- `modelo-busca-rosto` — Model `FaceSearch`
- `migrations` — Alterações em migrations
- `factory` — Factories para testes

### Funcionalidades Principais
- `autenticacao` — Autenticação e autorização
- `reconhecimento-facial` — `FaceRecognitionService`, processamento de rostos
- `processamento-foto` — `PhotoProcessingService`, jobs de processamento
- `carrinho` — `CartService`, gestão de carrinho
- `pagamentos` — Sistema de pagamentos
- `eventos` — Gestão de eventos
- `pedidos` — Gestão de pedidos e downloads

### UI & Frontend
- `componentes` — Componentes Livewire/Volt
- `formularios` — Forms do Livewire
- `views` — Templates Blade
- `tailwind` — Estilos CSS/Tailwind
- `assets` — Imagens, icons, brand

### Infra & DevOps
- `config` — Arquivos de configuração
- `jobs` — Queue jobs (`IndexPhotoFacesJob`, `ProcessEventPhotoJob`)
- `middleware` — Middleware HTTP
- `providers` — Service providers
- `logging` — Sistema de logging

## Exemplos de Commits Bem Formatados

### Feature
```
feat(reconhecimento-facial): implementa busca por similaridade de rostos

Adiciona algoritmo de comparação de embeddings para encontrar rostos
similares em um evento. Integra com FaceRecognitionService e adiciona
cache de resultados para melhor performance.

Closes #45
```

### Fix
```
fix(processamento-foto): corrige erro ao processar fotos com EXIF inválido

O processamento falhava quando metadados EXIF estavam corrompidos.
Agora implementa tratamento de exceção e log do erro sem interromper
o pipeline.

Closes #78
```

### Refactor
```
refactor(carrinho): simplifica lógica de cálculo de totais

Move cálculo para method dedicado no CartService, melhorando
testabilidade e legibilidade. Sem mudança de comportamento.
```

### Perf
```
perf(reconhecimento-facial): otimiza query de busca de fotos

Adiciona índice no banco em `photo_faces.embedding_hash` para
reduzir tempo de query de 2s para 150ms em conjunto de 100k fotos.
```

## Regras Obrigatórias

1. ✅ **Use sempre português-br** — type, escopo, assunto, corpo e rodapé
2. ✅ **Primeira linha máx. 72 caracteres** — tipo + escopo + assunto conciso
3. ✅ **Sem ponto final** na linha de assunto
4. ✅ **Descreva QUÊ e POR QUÊ** no corpo, não o COMO
5. ✅ **Inclua referências** no rodapé (`Closes #123`, `Refs #456`)
6. ✅ **Use lowercase** em tipo e escopo

## Regras PHP/Laravel Específicas

- **Migrations**: sempre incluir `migrations` no escopo
- **Models**: referenciar o nome do modelo quando relevante (`modelo-usuario`, não apenas `models`)
- **Serviços**: referenciar o serviço específico (`reconhecimento-facial`, `processamento-foto`)
- **Jobs**: incluir `jobs` ou o nome da classe (`IndexPhotoFacesJob`)
- **Testes**: sempre marcar com `test` e específico (`test(modelo-usuario)`)

## Checklist para Commit

Antes de fazer commit, verifique:

- [ ] Tipo é válido (feat/fix/refactor/perf/test/docs/style/chore/ci)
- [ ] Escopo é específico do FOTX
- [ ] Assunto em português-br, sem ponto final
- [ ] Primeira linha <= 72 caracteres
- [ ] Corpo explica motivação/contexto
- [ ] Rodapé inclui `Closes #XXX` se aplicável
- [ ] Código segue padrão PSR-12 (Laravel)
